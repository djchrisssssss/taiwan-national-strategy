## 1. 摘要

本報告記錄台灣國家戰略專案目前所有結構化量化資料點的驗證方法與結果。此版本的儲存庫共追蹤 180 筆參考來源、180 個可稽核 URL，以及分布於 16 個具統計章節中的 204 筆個別標註來源統計資料。所有資料點均標記為 `verified: true`，並附有相應的驗證說明；來源登錄簿也已從「假定有效」改為反映即時 HTTP 稽核結果的狀態分類。

> **備註（2026-03-30）**：本報告反映目前 17 章全文結構中，Ch01--Ch16 的統計資料層與新增的第 17 章綜整結論。bibliography / registry 補齊、citation integrity 驗證與重新執行即時 URL 稽核後的狀態，皆已體現在目前的 bibliography、statistics 與 source registry 中。

主要發現：

- **來源品質強，但集中度高。** 政府與報告型來源合計 156 / 180 筆（86.7%），整體證據基礎高度依賴機構型來源。
- **即時 URL 稽核已上線。** 2026-03-30 的稽核結果為 `reachable` 126 筆、`access_restricted` 27 筆、`missing` 15 筆、`other` 12 筆。
- **結構化資料完整性高。** 全部 204 筆統計資料皆通過 schema 驗證，且目前仍全部標記為 `verified: true`。
- **目前主要維護負擔在連結老化與存取限制，不在引用缺漏。** 驗證上的主要缺口來自 404、逾時、伺服器錯誤與受限存取頁面，而非缺少來源記錄。

## 2. 方法論

### 2.1 來源選擇標準

來源依據分層可靠性架構進行選擇：

| 層級 | 來源類別 | 選擇標準 |
|------|----------|----------|
| 第1層（最高） | 政府機構（台灣、美國、日本、聯合國） | 具機構權威性的官方統計 |
| 第2層 | 國際組織（IEA、FAO、UNCTAD、IISS） | 經同儕審查或編輯審核的資料 |
| 第3層 | 產業組織與企業申報（SIA、台積電投資人關係、ASML） | 經審計的財務資料及產業共識數據 |
| 第4層 | 研究機構與智庫（CSIS、RAND、史丹佛HAI） | 方法論透明的專家分析 |
| 第5層 | 新聞機構（路透社、彭博、DigiTimes） | 具編輯標準的調查報導 |
| 第6層 | 獨立追蹤資料庫（TeleGeography、Gerald C. Brown ADIZ追蹤器） | 方法論有據可查的公開來源情報 |

以下來源不予採用：（a）缺乏可辨識的機構背書；（b）無法與至少一個其他來源交叉驗證；（c）提出分析但未揭露方法論。

### 2.2 資料點驗證流程

每筆資料點經過四步驟驗證流程：

1. **來源辨識** -- 針對每項統計數據辨識主要權威來源（例如：營收數據來自台積電投資人關係、國防預算來自國防部）。
2. **交叉驗證** -- 在可能的情況下，將數據與至少一個次要來源比對。各統計JSON檔案中的 `verification_note` 欄位記錄了交叉驗證內容。
3. **情境合理性檢查** -- 評估每項數據與相關資料點的一致性（例如：台積電營收與台灣半導體產業總營收的對比；國防預算與GDP比例的對比）。
4. **文件記錄** -- 以結構化JSON格式記錄來源URL、存取日期及驗證說明。

### 2.3 JSON Schema驗證

所有統計檔案均符合 `data/schemas/statistics-schema.json` 定義的綱要。該綱要規範：

- 必填欄位：`id`、`metric`、`value`、`unit`、`date`、`source`（含 `name` 及 `url` 子欄位）
- ID格式：`^ch[0-9]{2}-[0-9]{3}$`
- 值類型：`number` 或 `string`（容納量化數值及質性描述）
- 選填欄位：`context`、`verified`、`verification_note`、`source.doi`、`source.accessed`

### 2.4 交叉引用方法論

參考文獻（`references/bibliography.json`）與來源登錄簿（`references/source-registry.json`）構成來源管理的雙主幹：

- **參考文獻**：記錄 180 個來源的後設資料，包含類型、作者、出版日期、URL、DOI（如有）、引用章節及驗證狀態。
- **來源登錄簿**：追蹤 URL 層級的驗證，包含狀態標記（`reachable`、`access_restricted`、`missing`、`other`）、最後檢查日期，以及各 URL 關於存取限制或穩定性風險的說明。
- **引用完整性驗證**：validator 現在會交叉檢查各章參考文獻與全文中的 `src-*` 引用，並在章節 reference table 提供明確對應時，比對 statistics JSON 的 `source.url` 是否能對到 registry 中的來源。

此二登錄簿與各章統計檔案之間的交叉引用，確保任何資料點均可追溯至其權威來源。
